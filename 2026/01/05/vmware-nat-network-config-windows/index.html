在VMWare-Centos上配置NAT网络

发表于2026-01-05|更新于2026-01-05|运维
|浏览量:

配置VMWare-NAT模式

1. 写在前面

各个设备之间的左右和职能大概是这样

角色 IP 职责
CentOS 虚拟机 192.168.200.10 内网主机
VMnet8 虚拟交换机 — 二层转发
VMware NAT 引擎 192.168.200.2 三层网关 + NAT
宿主机 VMnet8 网卡 192.168.200.1 管理 / 通信
宿主机真实网卡 真实 IP 外网出口
  1. VMnet8 本质是一台二层虚拟交换机，只根据 MAC 地址在虚拟机网卡、宿主机 VMnet8 网卡以及 NAT 引擎之间转发以太网帧，不参与任何三层决策；
  2. 真正承担“出口”角色的是 VMware 的 NAT 引擎，它作为 192.168.200.0/24 网段的三层网关，接收来自虚拟机的数据包后进行源地址转换（SNAT），把源 IP 从虚拟机地址改写为宿主机的真实 IP，再通过宿主机的物理网卡发送到外部网络，回包到达宿主机后再由 NAT 引擎依据映射表还原目标地址并经 VMnet8 转发回虚拟机；

2. 各个设备配置方式

  • 为虚拟机划分的子网，不能和宿主机同一个网段。查询宿主机的网络：

  • 查看虚拟网络编辑器的网段配置：

    编辑-虚拟网络编辑器

  • 再打开NAT设置，将网关设为 .2的 或 .254 ，一般不 设置为 .1 , 宿主机虚拟网卡 VMnet8 为.1

  • 设置centos的网络,后重启

  • 宿主机的VMnet8网卡的配置

3. 测试

  1. 在 CentOS 虚拟机中

    • default表示“默认路由”。当目标 IP 不匹配任何更具体路由规则时，数据包就走这条路由。
    • via 192.168.200.2 表示下一跳网关是 192.168.200.2 正是 VMware NAT 引擎,是虚拟机访问外部网络的唯一出口.
    • dev ens33 表示数据包从哪一块网卡发出去。
    • **proto static **这条路由的来源是 静态配置，意味着你在网络配置文件中显式配置了 GATEWAY。
    • metric 100 优先级，数值越小优先级越高。

  2. 分层连通性测试（由近到远）

  3. 验证“是否真的走 NAT，而不是走宿主机转发”

在虚拟机中执行：traceroute 8.8.8.8,正确现象应当是：第一跳：192.168.200.2,

  1. VMnet8 网卡 没有默认网关
